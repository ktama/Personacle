## 1. 背景と目的

大規模言語モデルによる対話AIは高品質だが、クラウドAPIへの依存によりプライバシーの懸念と継続的な利用コストが生じる。また、一般的な対話AIは会話のたびに記憶がリセットされ、ユーザーとの関係が積み重ならない。

本プロジェクトは、ローカルPC上のSLM(小規模言語モデル)で動作する「人格を持ったAIキャラクター(ペルソナ)」の育成アプリケーションを開発する。ペルソナは会話の内容を記憶として蓄積し、対話の経験によって性格や関係性が変化する。ユーザーとの1対1対話に加え、ペルソナ同士が自律的に会話する様子を観察でき、その会話もペルソナの記憶と成長に反映される。

完成した状態では、ユーザーは外部サービスに一切データを送信せずに、時間とともに個性が深まっていく複数のAIキャラクターを手元のPCで育てられる。本製品は将来、一般ユーザーへの公開を予定している。

**v0.2追補の目的**: v0.1.0では記憶と成長の仕組みは実装されたが、その変化は数値としてしか観測できず、育成の実感に乏しい。v0.2は、(1)ペルソナが時間の経過を認識しユーザーに働きかけること、(2)記憶が整理・統合されて熟成すること、(3)成長を可視化することにより、記憶と成長を体感できるものにする。あわせて初版で対象外とした「ユーザー+複数ペルソナのグループチャット」と、自律会話の停滞への対処を対象に加える。

## 2. 用語定義

| 用語 | 定義 |
| --- | --- |
| SLM | Small Language Model。一般消費者向けPCでローカル実行できる規模の言語モデル |
| ペルソナ | 本システムが管理するAIキャラクターの単位。名前・人格プロファイル・記憶・会話履歴を持つ |
| 人格プロファイル | ペルソナの性格・口調・価値観・他者への関係性を表すデータ。対話経験によって更新される |
| 記憶 | 会話から抽出され、ペルソナごとに永続保存される情報の単位。発生日時と出所(どの会話から生じたか)を持つ |
| 想起 | 進行中の会話に関連する記憶を取り出し、応答生成に反映すること |
| セッション | 開始から終了までの一続きの会話。1対1対話・自律会話・グループチャット(v0.2)の3種類がある |
| 自律会話 | ユーザーの発話なしに、2体以上のペルソナが交互に発話を続けるセッション |
| 推論エンジン | ローカルPC上でSLMを実行し、テキスト生成を行う外部ソフトウェア |
| グループチャット | ユーザーと2体以上のペルソナが参加するセッション(v0.2) |
| 話しかけ | ユーザーの発話に先立って、ペルソナ側から発せられる発話(v0.2) |
| 統合記憶 | 複数の記憶を要約・一般化して生成された記憶。統合元の記憶への参照を持つ(v0.2) |
| ムード | セッションの内容によって変動し、時間経過で平常に戻る短期的な感情状態。人格プロファイル(長期)とは区別される(v0.2) |
| 日記 | 対話があった日について、ペルソナ自身の視点で生成される振り返りの文章(v0.2) |
| 停滞 | 自律会話で、同じ趣旨の発話が繰り返されて会話が進まなくなる状態(v0.2) |

## 3. 前提と仮定

**前提(確定している条件)**:

- 推論はユーザーのローカルPC上で完結する(発注者の確定要望)
- 利用者は1台のPCにつき1人(シングルユーザー)
- 開発および初版の動作対象は Windows 11
- 対話はテキストのみ(音声・画像は扱わない)

**仮定(未確認だが置いている仮定)**:

- (仮定) 初版の対象ユーザーは、ローカルLLM実行環境を手順書に従ってセットアップできる技術リテラシーを持つ(一般消費者向けの完全自動セットアップは初版では求めない)
- (仮定) 対話言語は日本語を主とする。他言語の品質は保証対象外
- (仮定) 推奨動作環境はメモリ16GB以上のPCとし、GPUは必須としない
- (仮定) 将来公開時も、本要件の範囲はシングルユーザー・ローカル完結のまま変わらない(クラウド化は別プロジェクトとする)

**v0.2追補の前提と仮定**:

- v1.1の前提・制約(ローカル完結、シングルユーザー、Windows 11、日本語、メモリ16GB・GPUなし)は変更しない。NFR-01〜07はv0.2の新機能にもそのまま適用される
- v0.2の新機能が行う推論・埋め込み計算は、既存の推論エンジン接続(FR-16)の範囲で行い、新たな外部依存を追加しない
- (仮定) ムードはセッションをまたいで持続し、対話のない時間の経過によって平常へ回帰する。現在のムードはユーザーに表示する(隠しパラメータにしない)
- (仮定) 日記は読み物であり、想起(FR-09)には使わない。想起は引き続き記憶が担う
- (仮定) グループチャットの参加ペルソナ数の上限は自律会話(FR-19)と同水準とする(具体値は未解決事項9-12)

## 4. スコープ

### 4.1 対象範囲

1. ペルソナの作成・閲覧・編集・削除
2. ユーザーとペルソナの1対1テキスト対話(GUI)
3. ペルソナ同士(2体)の自律会話と、その観察・制御
4. 記憶システム(会話からの記憶生成、保存、想起、応答への反映)
5. 人格の成長(性格・関係性の対話経験による変化)
6. 記憶・人格プロファイルのユーザーによる閲覧・編集
7. ローカル推論エンジンとの接続と使用モデルの設定
8. 全データの永続化(再起動後の保持)
9. 時間経過の認識と、ペルソナからのアプリ内の話しかけ (v0.2)
10. 記憶の自動的な整理・統合と、統合の由来の閲覧 (v0.2)
11. ムード(短期的な感情状態)の保持・応答への反映・閲覧 (v0.2)
12. ペルソナの日記の生成と閲覧 (v0.2)
13. 成長の可視化(性格・親密度の推移表示)とペルソナ関係図 (v0.2)
14. 記憶の検索・絞り込み (v0.2)
15. ユーザーと複数ペルソナのグループチャット (v0.2)
16. 自律会話の停滞の検出と話題転換 (v0.2)

### 4.2 対象外 (non-goals)

- **ユーザー+複数ペルソナのグループチャット**: ~~発言順制御の複雑さが大きいため初版では見送り、将来拡張とする~~ → v0.2で対象範囲に変更(対象範囲15)
- **音声対話・アバター/立ち絵表示**: テキスト対話の体験を先に確立するため対象外
- **クラウドAPI利用・クラウド同期・マルチユーザー対応**: ローカル完結・プライバシー保護が本製品の核であるため対象外
- **SLM本体のファインチューニング**: 人格はプロンプトと記憶データで実現する。学習基盤の構築はコストに見合わないため対象外
- **Windows 以外のOS対応**: 初版は開発環境と同じ Windows に限定し、他OSは公開時に再検討する
- **推論エンジン自体の同梱・自動インストール**: 初版はユーザーが別途導入した推論エンジンに接続する方式とし、同梱は公開時に再検討する

**v0.2で追加する対象外**:

- **OS通知・常駐によるアプリ外からの話しかけ**: 通知の煩わしさと常駐化の実装コストが体験向上に見合わないため、話しかけはアプリ内(チャット画面を開いた時)に限定する
- **人格破綻(EC-11)の自動検出**: SLM自身に矛盾判定をさせると誤検出が多くなる懸念があり、検証コストが大きいため次版以降で再検討する
- **記憶統合のユーザー承認フロー**: 承認作業が育成体験を作業化するため対象外。誤統合の是正は元記憶の閲覧(FR-23)と記憶の編集・削除(FR-11)で担保する

## 5. 機能要件

### 5.1 ペルソナ管理

### FR-01: ペルソナの作成 [優先度: Must]

**要件**: ユーザーは、名前と人格の初期設定(性格の記述、口調、価値観、自己紹介文)を入力して新しいペルソナを作成できる。作成時点で人格プロファイルが初期化される。

**受け入れ基準**:

- [ ] 名前と性格の記述を入力して作成を実行すると、一覧に新しいペルソナが表示される
- [ ] 作成直後のペルソナと対話すると、初期設定に沿った口調・内容の応答が返る(第三者が初期設定文と応答を見比べて矛盾がないと判定できる)
- [ ] 名前が空の場合は作成できず、名前が必須である旨が表示される

### FR-02: ペルソナの一覧と詳細閲覧 [優先度: Must]

**要件**: ユーザーは、作成済みペルソナの一覧を表示し、各ペルソナの名前・初期設定・最終対話日時を確認できる。

**受け入れ基準**:

- [ ] 作成済みの全ペルソナが一覧に表示される
- [ ] 一覧から1体を選ぶと、そのペルソナの初期設定と最終対話日時が表示される

### FR-03: ペルソナの初期設定の編集 [優先度: Should]

**要件**: ユーザーは、作成後のペルソナの名前と初期設定を編集できる。編集は初期設定のみを変更し、蓄積済みの記憶と成長済みの人格プロファイルは保持される。

**受け入れ基準**:

- [ ] 初期設定を編集して保存すると、次の対話から編集後の設定が反映される
- [ ] 編集の前後で、そのペルソナの記憶の件数と人格プロファイルの成長分が変化しない

### FR-04: ペルソナの削除 [優先度: Must]

**要件**: ユーザーは、ペルソナを削除できる。削除するとそのペルソナの人格プロファイル・記憶・会話履歴は復元できない。削除前に確認を求める。

**受け入れ基準**:

- [ ] 削除操作を実行すると確認表示が出て、承認した場合のみ削除される
- [ ] 削除後、そのペルソナは一覧に表示されず、アプリを再起動しても復活しない
- [ ] 削除の確認表示に「復元できない」旨が明記されている

### 5.2 対話

### FR-05: ユーザーとの1対1対話 [優先度: Must]

**要件**: ユーザーは、選択したペルソナとテキストで対話できる。ペルソナの応答は、人格プロファイル・想起された記憶・当該セッション内の直前のやり取りを反映して生成される。応答は生成の進行に合わせて逐次表示される。

**受け入れ基準**:

- [ ] メッセージを送信すると、選択中のペルソナからの応答が表示される
- [ ] 応答の文字列が、生成完了を待たずに順次画面に現れる
- [ ] 同一セッション内で3往復前に伝えた情報(例: ユーザーの好物)を尋ねると、応答にその情報が含まれる

### FR-06: 会話履歴の保存と閲覧 [優先度: Must]

**要件**: 全セッションの発話(発話者・本文・日時)は保存され、ユーザーはペルソナごと・セッションごとに過去の会話を閲覧できる。

**受け入れ基準**:

- [ ] 対話終了後にアプリを再起動しても、当該セッションの全発話が閲覧できる
- [ ] ペルソナを選ぶと、そのペルソナが参加した過去セッションの一覧が新しい順に表示される

### FR-07: 応答生成のキャンセル [優先度: Must]

**要件**: ユーザーは、応答の生成中にキャンセルできる。キャンセル時点までに生成された部分は発話として履歴に残る。

**受け入れ基準**:

- [ ] 生成中にキャンセル操作をすると、5秒以内に生成が止まり入力欄が再び使える状態になる
- [ ] キャンセルされた発話は、途中までの本文とともに履歴に残る

### 5.3 記憶

### FR-08: 会話からの記憶の生成 [優先度: Must]

**要件**: システムは、セッション中またはセッション終了時に、会話に含まれる情報(相手について知った事実、出来事、交わした約束、抱いた感想)をペルソナの記憶として生成・保存する。各記憶は発生日時と出所セッションを持つ。

**受け入れ基準**:

- [ ] ユーザーが自分に関する事実(例: 職業)を伝えたセッションの終了後、その事実に対応する記憶が記憶一覧に存在する
- [ ] 各記憶に発生日時と出所セッションへの参照が記録されている

### FR-09: 記憶の想起と応答への反映 [優先度: Must]

**要件**: ペルソナは応答を生成するとき、進行中の話題に関連する過去の記憶を想起し、応答に反映する。過去のセッションで記憶した情報について尋ねられた場合、記憶に基づいて答えられる。

**受け入れ基準**:

- [ ] あるセッションでユーザーの好物を伝え、セッションを終了し、アプリを再起動した後の新しいセッションで好物を尋ねると、応答に記憶した好物が含まれる
- [ ] 記憶にない事柄を尋ねた場合に、記憶している体で断定する応答ではなく、知らない・覚えていない旨の応答が返る(10回試行中7回以上)

### FR-10: 記憶の閲覧 [優先度: Must]

**要件**: ユーザーは、ペルソナごとの記憶の一覧を、本文・発生日時・出所とともに閲覧できる。

**受け入れ基準**:

- [ ] ペルソナを選んで記憶一覧を開くと、保存済みの全記憶が発生日時付きで表示される

### FR-11: 記憶の編集と削除 [優先度: Should]

**要件**: ユーザーは、個々の記憶の本文を編集、または記憶を削除できる。誤った記憶や不要な記憶をユーザーが直接是正する手段を提供する。

**受け入れ基準**:

- [ ] 記憶を削除した後の対話で、削除した情報を尋ねると、知らない旨の応答が返る(10回試行中7回以上)
- [ ] 記憶の本文を編集すると、以後の想起では編集後の内容が使われる

### 5.4 人格の成長

### FR-12: 人格プロファイルの経験による更新 [優先度: Must]

**要件**: ペルソナの人格プロファイルは、セッションでの経験に基づいて更新される。更新対象には、少なくとも相手(ユーザーおよび他ペルソナ)ごとの関係性(親密度・印象)と、性格傾向が含まれる。1回のセッションによる変化量には上限があり、単一セッションで人格が急変しない。

**受け入れ基準**:

- [ ] 好意的な内容の1対1セッションを10回行った後、ユーザーへの親密度の値が開始前より上昇している
- [ ] 1回のセッションの前後で、各性格傾向の変化量が仕様で定めた1セッションあたり上限値を超えない
- [ ] 人格プロファイルの更新はセッション単位で記録され、いつどのセッションで何が変化したかを後から確認できる

### FR-13: 人格プロファイルの閲覧 [優先度: Must]

**要件**: ユーザーは、ペルソナの現在の人格プロファイル(性格傾向、相手ごとの関係性)と、その変化の履歴を閲覧できる。

**受け入れ基準**:

- [ ] ペルソナの詳細画面で、現在の性格傾向と相手ごとの関係性が表示される
- [ ] 過去の変化履歴(日時・変化した項目・変化前後の値)が閲覧できる

### 5.5 自律会話

### FR-14: 自律会話セッションの実行 [優先度: Must]

**要件**: ユーザーは、2体のペルソナと会話のテーマを指定して自律会話セッションを開始できる。ペルソナは交互に発話し、ユーザーは会話を観察できる。会話は指定ターン数(既定値と上限を設ける)に達すると自動終了し、ユーザーはいつでも手動停止できる。

**受け入れ基準**:

- [ ] 2体とテーマを指定して開始すると、両ペルソナの発話が交互に画面へ追加されていく
- [ ] 各ペルソナの発話が、それぞれの人格プロファイルと記憶を反映している(第三者が2体の初期設定と発話を見比べ、どちらの発話か判別できる)
- [ ] 指定ターン数に達すると新しい発話が生成されず、セッションが終了状態になる
- [ ] 手動停止すると、次の発話の生成が始まる前に会話が止まる

### FR-15: 自律会話の記憶・人格への反映 [優先度: Must]

**要件**: 自律会話の内容は、参加した各ペルソナの記憶生成(FR-08)と人格プロファイル更新(FR-12)の対象になる。ペルソナは自律会話で知った情報を、後のユーザーとの1対1対話で参照できる。

**受け入れ基準**:

- [ ] ペルソナAがペルソナBとの自律会話で話した情報を、後でユーザーがペルソナBに尋ねると、応答にその情報が含まれる
- [ ] 自律会話セッションの終了後、参加ペルソナ間の関係性の値が更新履歴に記録されている

### 5.6 実行基盤

### FR-16: 推論エンジンの接続設定 [優先度: Must]

**要件**: ユーザーは、ローカルPC上の推論エンジンへの接続先と使用モデルを設定画面で指定・変更できる。接続確認の手段を提供する。

**受け入れ基準**:

- [ ] 接続先と使用モデルを設定して接続確認を実行すると、成否が表示される
- [ ] 使用モデルを変更した後も、既存のペルソナ・記憶・会話履歴がそのまま使える

### FR-17: データの永続化 [優先度: Must]

**要件**: ペルソナ・人格プロファイル・記憶・会話履歴・設定は、ローカルストレージに保存され、アプリと推論エンジンの再起動後も保持される。

**受け入れ基準**:

- [ ] ペルソナ作成・対話・成長の発生後にアプリとPCを再起動し、全データ(ペルソナ、記憶、履歴、人格プロファイル、設定)が再起動前と一致する

### FR-18: ペルソナのエクスポートとインポート [優先度: Could]

**要件**: ユーザーは、ペルソナ1体分のデータ(初期設定・人格プロファイル・記憶。会話履歴を含むかは選択可)をファイルに書き出し、別環境の本アプリに取り込める。

**受け入れ基準**:

- [ ] エクスポートしたファイルを別のPC上の本アプリでインポートすると、同じ初期設定・人格プロファイル・記憶を持つペルソナが再現される

### FR-19: 3体以上の自律会話 [優先度: Could]

**要件**: 自律会話セッション(FR-14)に3体以上のペルソナを参加させられる。発話順の制御方式は設計に委ねる。

**受け入れ基準**:

- [ ] 3体を指定して開始すると、3体全員の発話がセッション中に1回以上現れる

### 5.7 時間経過の認識と話しかけ (v0.2)

### FR-20: 応答への時間経過の反映 [優先度: Must]

**要件**: ペルソナは、相手との前回のセッションからの経過時間を認識し、新しいセッションでの応答に反映する。経過時間の長短に応じた言及の閾値(何時間以上空いたら「間が空いた」と扱うか)は設計で確定し、設定として保持する。

**受け入れ基準**:

- [ ] 最終対話から閾値以上(例: 72時間)空けて新しいセッションを開始し挨拶すると、最初の応答に期間の経過と整合する言及が含まれる(10回試行中7回以上)
- [ ] 最終対話から1時間以内に新しいセッションを開始した場合、最初の応答に長期間の経過を示す言及(「久しぶり」に類する表現)が含まれない(10回試行中7回以上)

### FR-21: ペルソナからの話しかけ [優先度: Must]

**要件**: ユーザーがペルソナとのチャット画面を開いたとき、ユーザーの入力に先立ってペルソナ側から最初の発話(話しかけ)が生成・表示される。話しかけの内容は、そのペルソナの人格プロファイル・記憶・前回からの経過時間を反映する。ユーザーは設定で話しかけを無効化できる。

**受け入れ基準**:

- [ ] 話しかけが有効な状態でチャット画面を開くと、ユーザーの入力なしにペルソナの発話が表示される
- [ ] 前回までのセッションで交わした約束の記憶がある場合、話しかけにその約束への言及が含まれる(10回試行中7回以上)
- [ ] 設定で話しかけを無効にすると、チャット画面を開いてもペルソナの発話は生成されず、ユーザーの送信後に初めて応答する

### 5.8 記憶の整理・統合 (v0.2)

### FR-22: 記憶の自動統合 [優先度: Must]

**要件**: システムは、同一の話題に関する複数の記憶を要約・一般化した統合記憶を自動で生成する。統合された元の記憶はアーカイブ状態(想起の対象外・閲覧は可能)になり、削除はされない。統合の実行はユーザーの操作を必要とせず、1対1対話の応答生成を妨げないタイミングで行う。統合のトリガー条件(対象件数、類似の判定)は設計で確定する。

**受け入れ基準**:

- [ ] 同一の話題に関する記憶が複数(5件以上)ある状態で統合が実行されると、内容を要約した統合記憶が1件生成され、元の記憶はアーカイブ状態になる
- [ ] 統合の実行後の対話で当該話題について尋ねると、統合前と同じく記憶に基づいた応答が返る(10回試行中7回以上)
- [ ] 統合の実行前後で、1対1対話の応答性能がNFR-01の基準を満たし続ける

### FR-23: 統合の由来の閲覧 [優先度: Must]

**要件**: ユーザーは、統合記憶について、統合元となった記憶の一覧を閲覧できる。また元の記憶側からも、どの統合記憶に統合されたかを確認できる。

**受け入れ基準**:

- [ ] 統合記憶を選ぶと、統合元の記憶の一覧(本文・発生日時)が表示される
- [ ] アーカイブされた元の記憶を閲覧すると、統合先の統合記憶が表示される

### 5.9 ムード (v0.2)

### FR-24: ムードの保持と応答への反映 [優先度: Should]

**要件**: ペルソナは、セッションの内容によって変動する短期的なムードを持ち、応答の口調・内容に反映する。ムードはセッションをまたいで持続し、対話のない時間の経過によって平常へ回帰する。ムードの変動は人格プロファイル(性格傾向・親密度)の値を直接変更しない。ムードの表現形式と回帰までの期間は設計で確定する。

**受け入れ基準**:

- [ ] ムードを大きく変動させる内容のセッション(例: 強い叱責)の直後の応答と平常時の応答を第三者が見比べて、トーンの違いを判別できる(10回試行中7回以上)
- [ ] 対話のない期間が設計で定めた回帰期間を超えると、ムードは平常値に戻っている
- [ ] ムードが変動したセッションの前後で、性格傾向と親密度の値はFR-12の更新分以外に変化しない

### FR-25: ムードの閲覧 [優先度: Should]

**要件**: ユーザーは、ペルソナの現在のムードと、直近の変動要因(どのセッションで変動したか)を閲覧できる。

**受け入れ基準**:

- [ ] ペルソナの詳細画面で現在のムードが表示される
- [ ] ムードの直近の変動について、変動元のセッションへの参照が表示される

### 5.10 日記 (v0.2)

### FR-26: 日記の生成 [優先度: Should]

**要件**: システムは、ペルソナが対話(1対1・自律会話・グループチャット)を行った日について、そのペルソナ自身の視点で書かれた日記を1日1件生成する。対話がなかった日の日記は生成しない。日記は閲覧用であり、想起(FR-09)には使わない。

**受け入れ基準**:

- [ ] 1件以上のセッションを行った日について、そのペルソナの日記が1件生成される
- [ ] 対話がなかった日の日記は生成されない
- [ ] 日記の内容がその日のセッション内容と整合する(第三者が読み比べて矛盾がないと判定できる)

### FR-27: 日記の閲覧 [優先度: Should]

**要件**: ユーザーは、ペルソナごとの日記を日付の新しい順で一覧・閲覧できる。

**受け入れ基準**:

- [ ] ペルソナを選んで日記一覧を開くと、生成済みの日記が日付の新しい順に表示される

### 5.11 記憶の検索と成長の可視化 (v0.2)

### FR-28: 記憶の検索・絞り込み [優先度: Must]

**要件**: ユーザーは、ペルソナの記憶一覧(FR-10)をキーワードで検索し、記憶の種別(事実・出来事・約束・感想)およびアーカイブ状態(通常のみ / アーカイブを含む)で絞り込める。

**受け入れ基準**:

- [ ] キーワードを入力すると、本文にそのキーワードを含む記憶のみが表示される
- [ ] 種別とアーカイブ状態の絞り込みを指定すると、条件に合致する記憶のみが表示される
- [ ] 該当する記憶が0件の場合、0件である旨が表示される

### FR-29: 成長ダッシュボード [優先度: Should]

**要件**: ユーザーは、ペルソナの性格傾向(5軸)と相手ごとの親密度について、値の推移を時系列のグラフで閲覧できる。表示される値は変化履歴(FR-13)と一致する。

**受け入れ基準**:

- [ ] ペルソナを選ぶと、性格5軸それぞれの値の推移が時系列で表示される
- [ ] 相手を選ぶと、その相手への親密度の推移が時系列で表示される
- [ ] グラフ上の任意の時点の値が、FR-13の変化履歴の記録と一致する

### FR-30: ペルソナ関係図 [優先度: Could]

**要件**: ユーザーは、全ペルソナとユーザーを含む関係の図を閲覧できる。図上では関係の強さ(親密度)が視覚的に区別でき、図から各ペルソナの詳細(FR-13)へ移動できる。

**受け入れ基準**:

- [ ] 関係図に全ペルソナとユーザーが表示され、親密度の高低が図上で区別できる
- [ ] 図上のペルソナを選ぶと、そのペルソナの人格プロファイル閲覧(FR-13)へ移動する

### 5.12 グループチャット (v0.2)

### FR-31: グループチャットの実行 [優先度: Must]

**要件**: ユーザーは、2体以上のペルソナを指定してグループチャットを開始できる。ユーザーが発話すると、発話の内容と文脈に基づいて選ばれた1体のペルソナが応答する。各ペルソナの応答は、それぞれの人格プロファイルと記憶を反映する。

**受け入れ基準**:

- [ ] 2体を指定して開始しユーザーが発話すると、参加ペルソナのうち1体が応答する
- [ ] 応答したペルソナの発話が、そのペルソナの人格プロファイルと記憶を反映している(第三者が初期設定と発話を見比べ、どのペルソナの発話か判別できる)
- [ ] 特定のペルソナに宛てない発話を参加数×3回行ったとき、全参加ペルソナが1回以上応答している(10回試行中7回以上)

### FR-32: 応答ペルソナの指名 [優先度: Must]

**要件**: ユーザーは、グループチャットで応答するペルソナを指名して発話できる。指名した場合は、指名されたペルソナが応答する。

**受け入れ基準**:

- [ ] ペルソナを指名して発話すると、指名されたペルソナが応答する(毎回)

### FR-33: ペルソナ間の連鎖発話 [優先度: Should]

**要件**: グループチャットで、あるペルソナの応答に対して、ユーザーの発話を待たずに別のペルソナが続けて発話することがある。連鎖する発話数には上限(既定2発話)を設け、連鎖中でもユーザーはいつでも発話できる。

**受け入れ基準**:

- [ ] 参加数×5回のユーザー発話を行うセッションで、ペルソナ間の連鎖発話が1回以上発生する(10回試行中7回以上)
- [ ] 連鎖発話が既定の上限を超えて続かない
- [ ] 連鎖中にユーザーが発話すると、次の連鎖発話より先にユーザー発話への応答が行われる

### FR-34: グループチャットの記憶・人格への反映 [優先度: Must]

**要件**: グループチャットの内容は、参加した各ペルソナの記憶生成(FR-08)と人格プロファイル更新(FR-12)の対象になる。反映の範囲はユーザーへの関係性と、参加ペルソナ相互の関係性の両方を含む。

**受け入れ基準**:

- [ ] グループチャットでユーザーが伝えた情報を、後の1対1対話で参加していたペルソナに尋ねると、応答にその情報が含まれる
- [ ] セッション終了後、参加ペルソナのユーザーへの関係性と参加ペルソナ相互の関係性の更新が変化履歴に記録されている

### 5.13 自律会話の品質 (v0.2)

### FR-35: 停滞の検出と話題転換 [優先度: Should]

**要件**: システムは、自律会話セッション中の停滞(同じ趣旨の発話の繰り返し)を検出し、検出した場合は話題を転換して会話を継続させる。検出の判定方式は設計で確定する。話題転換の発生はセッション履歴から確認できる。

**受け入れ基準**:

- [ ] 停滞が検出されると、以後の発話の話題が転換され、セッションは指定ターン数まで継続する
- [ ] 話題転換が発生したことと発生位置が、セッション履歴の閲覧で確認できる
- [ ] 停滞が検出されないセッションでは話題転換が挿入されない

## 6. 非機能要件

### NFR-01: 応答性能 [Must]

推奨動作環境(3章の仮定を参照)において、1対1対話の応答は、送信から最初の文字の表示開始まで10秒以内(10回試行の中央値)。1発話の生成完了までは60秒以内(同中央値)。

### NFR-02: オフライン動作 [Must]

モデルファイルと推論エンジンの導入完了後は、ネットワーク接続を切断した状態で本書の全Must機能が動作する。

### NFR-03: プライバシー [Must]

会話・記憶・人格プロファイル・設定の全データはローカルPC内にのみ保存される。ユーザーの明示的なエクスポート操作(FR-18)を除き、本アプリはこれらのデータを外部ネットワークに送信しない。

### NFR-04: 容量 [Should]

1ペルソナあたり記憶10,000件・累計発話50,000件、ペルソナ20体までの規模で、NFR-01の応答性能と各画面の一覧表示(3秒以内)を維持する。

### NFR-05: データ保全 [Must]

アプリまたはPCの強制終了が起きても、直前のセッションまでに保存済みのデータは失われず、破損しない。強制終了時に生成途中だった応答・記憶は失われてよい。

### NFR-06: ログ [Should]

推論エンジンとの通信エラー、データ保存の失敗、アプリ内部のエラーは、発生日時・原因とともにローカルのログファイルに記録される。ログには会話本文を含めない設定を既定とする。

### NFR-07: セットアップ [Should]

同梱の導入手順書のみを参照して、対象ユーザー(3章の仮定に定めた技術リテラシーを持つユーザー)が推論エンジンの導入から初回対話まで到達できる。

### NFR-08: v0.2追加画面の表示性能 [Should] (v0.2)

NFR-04と同じ規模(1ペルソナあたり記憶10,000件・累計発話50,000件・ペルソナ20体)において、記憶の検索(FR-28)・成長ダッシュボード(FR-29)・関係図(FR-30)・日記一覧(FR-27)の表示が、操作から3秒以内(10回試行の中央値)に完了する。

### NFR-09: 話しかけの応答性能 [Must] (v0.2)

推奨動作環境において、チャット画面を開いてから話しかけ(FR-21)の最初の文字の表示開始まで10秒以内(10回試行の中央値)。NFR-01の1対1対話と同水準とする。

**該当なしと判断した観点**: 可用性(常時稼働サービスではないローカルアプリのため)、セキュリティの認証・認可(シングルユーザー・ローカル完結のため。ただしNFR-03のプライバシーで担保)、アクセシビリティ(初版では対象ユーザーを限定するため。公開時に再検討)、移行性(新規開発で既存データがないため。将来のデータ形式変更は未解決事項9-6参照)。

## 7. エッジケース・例外時の振る舞い

### EC-01: 初回起動(ペルソナ0件)

状況: アプリを初めて起動し、ペルソナが1体もない。
期待: 空の画面ではなく、ペルソナ作成へ誘導する案内が表示され、案内から作成(FR-01)を開始できる。

### EC-02: 推論エンジンに接続できない

状況: 推論エンジンが未起動、または接続設定が誤っている状態で対話を試みる。
期待: 接続できない旨と確認すべき事項(推論エンジンの起動、接続設定)が表示され、設定画面(FR-16)へ移動できる。入力済みのメッセージは消えずに残る。

### EC-03: 生成中の強制終了

状況: 応答または記憶の生成中にアプリやPCが強制終了する。
期待: 再起動後、保存済みのデータは失われない(NFR-05)。生成途中の応答・記憶は破棄され、当該セッションは閲覧可能な状態で残る。

### EC-04: 同名ペルソナの作成

状況: 既存ペルソナと同じ名前で新規作成しようとする。
期待: 同名である旨の警告が表示される。ユーザーが承認すれば作成できる(内部的には別個体として区別される)。

### EC-05: 入力上限の超過

状況: 1回のメッセージとして上限を超える長さのテキストを送信しようとする。
期待: 上限文字数(値は設計で確定し、画面に明示する)を超える場合は送信できず、超過している旨と上限値が表示される。

### EC-06: 記憶件数の上限到達

状況: あるペルソナの記憶件数がNFR-04の上限に達する。
期待: 対話と新しい記憶の保存は継続できる。保持する記憶の取捨方式は設計に委ねるが、ユーザーが直近のセッションで伝えた情報は優先的に保持される。

### EC-07: 削除されたペルソナに関する他ペルソナの記憶

状況: ペルソナAを削除したとき、ペルソナBがAとの自律会話で得た記憶を持っている。
期待: Bの記憶は削除されず保持される(Bにとっての過去の経験として残る)。Bの人格プロファイル中のAへの関係性データも履歴として閲覧できる。

### EC-08: 同一ペルソナの並行セッション

状況: あるペルソナが自律会話に参加中に、同じペルソナとの1対1対話を開始しようとする(またはその逆)。
期待: 1体のペルソナが同時に参加できるセッションは1つとし、2つ目の開始は参加中である旨の表示とともに拒否される。

### EC-09: 空メッセージの送信

状況: 空文字または空白のみのメッセージを送信しようとする。
期待: 送信は実行されない。

### EC-10: 特殊文字を含む入力

状況: 絵文字・サロゲートペア・改行を含むメッセージや名前を入力する。
期待: 入力内容が欠落・文字化けせずに保存・表示される。制御文字は保存前に除去される。

### EC-11: ペルソナ設定と矛盾する応答(人格の破綻)

状況: SLMの生成品質により、ペルソナが自身の設定・記憶と矛盾する発話をする。
期待: システムによる自動検出は初版の対象外とする。ユーザーが記憶の編集・削除(FR-11)と人格プロファイルの閲覧(FR-13)で状態を確認・是正できることをもって担保する。

### EC-12: 自律会話の停滞

状況: 自律会話で同じ内容の繰り返しが続く。
期待: ~~自動検出は初版の対象外とする。ターン数上限による自動終了(FR-14)と手動停止で担保する。~~ → v0.2でFR-35(停滞の検出と話題転換)により対処する。ターン数上限と手動停止も引き続き有効。

### EC-13: 話しかけ時に推論エンジンへ接続できない (v0.2)

状況: 話しかけ(FR-21)が有効な状態でチャット画面を開いたが、推論エンジンに接続できない。
期待: 話しかけは表示されず、エラーダイアログも出さずに通常の入力可能状態になる。ユーザーがメッセージを送信した時点でEC-02の振る舞いに従う。

### EC-14: 短時間での画面の開き直しによる話しかけの重複 (v0.2)

状況: 同じペルソナとのチャット画面を短時間に何度も開閉する。
期待: 話しかけで開始したセッションが継続している間は、画面を開き直しても新たな話しかけは生成されない。

### EC-15: 記憶統合の処理中の強制終了 (v0.2)

状況: 記憶の統合(FR-22)の実行中にアプリまたはPCが強制終了する。
期待: 元の記憶は失われない。統合は再起動後に再実行され、同じ元記憶の組から統合記憶が二重に生成されない。

### EC-16: 統合記憶の削除 (v0.2)

状況: ユーザーが統合記憶を削除(FR-11)する。
期待: 削除時に、アーカイブ済みの元記憶を想起の対象に戻すかどうかの確認が表示され、ユーザーが選択できる。

### EC-17: 日付をまたぐセッションの日記 (v0.2)

状況: 23時に開始し翌0時すぎに終了したセッションがある日の日記を生成する。
期待: そのセッションはセッション開始日の日記の対象になる。同じセッションが2日分の日記に重複して反映されない。

### EC-18: システム時計の巻き戻し (v0.2)

状況: ユーザーがPCのシステム時計を過去に変更した後に対話する。
期待: 前回からの経過時間が負になる場合、時間経過への言及(FR-20)は行われず、通常の応答になる。保存済みデータの日時は変更されない。

### EC-19: グループチャットとセッションの排他 (v0.2)

状況: グループチャットに参加中のペルソナで、1対1対話または自律会話を開始しようとする(またはその逆)。
期待: EC-08と同じ扱いとする。1体のペルソナが同時に参加できるセッションは1つであり、2つ目の開始は参加中である旨の表示とともに拒否される。

### EC-20: 話しかけ・日記・統合の対象データ不足 (v0.2)

状況: 作成直後で記憶が0件のペルソナのチャット画面を開く。または記憶が統合のトリガー条件に満たない。
期待: 話しかけは初期設定のみに基づいて生成される(エラーにならない)。統合は実行されず、エラーも記録されない。

**チェックリスト判定の記録**: A-1 入力・データ=該当あり(EC-04, 05, 09, 10。v0.2追加: EC-20、FR-28の0件表示)。A-2 状態・タイミング=該当あり(EC-01, 03, 08。v0.2追加: EC-14, 15, 19)。A-3 権限・利用者=該当なし(シングルユーザーで権限概念がないため)。A-4 外部依存=該当あり(EC-02。v0.2追加: EC-13。依存は推論エンジンのみ)。A-5 時間・地域=v1.1では該当なしだったが、v0.2の時間経過認識と日記の導入により該当ありに変更(EC-17, 18)。A-6 ライフサイクル=該当あり(FR-04, EC-06, 07。v0.2追加: EC-16。移行は未解決事項9-6)。

## 8. 制約

- 推論はユーザーのローカルPC上で完結すること(発注者の確定要望。クラウドAPIの利用は不可)
- 応答生成に用いるモデルは、推奨動作環境(メモリ16GB・GPUなし)で実行可能な規模のSLMであること
- 初版の動作対象OSは Windows 11 であること(開発環境による)
