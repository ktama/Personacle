# design-notes: Personacle 設計 作業メモ

入力: docs/requirements.md v0.1 (FR-01〜19, NFR-01〜07, EC-01〜12)

## Phase 1: 論点リスト

1. **UI形態と技術スタック** (要件 未解決9-1, NFR-07, 将来公開) — デスクトップアプリ / ローカルWeb / スタック選定
2. **推論エンジンとの接続方式** (未解決9-2, FR-16, NFR-01/02) — 接続プロトコルの標準と推奨エンジン
3. **全体構成** (A-1) — 単一プロセスかフロント/バック分離か。ADR-01に従属
4. **データ永続化方式** (FR-17, NFR-04/05, A-2) — SQLite / ファイル群 / 併用
5. **記憶の想起方式** (FR-09, NFR-01/04, EC-06) — 埋め込み類似検索 / 全文検索 / ハイブリッド
6. **記憶生成・人格更新のタイミング** (FR-08/12, NFR-01, EC-03) — 逐次 / セッション終了時 / バックグラウンド
7. **人格プロファイルの表現** (FR-12/13, 未解決9-4) — 数値パラメータ / 自由記述 / ハイブリッド。履歴の持ち方(追記イベント+現在値)
8. **自律会話の実行制御** (FR-14/15/19, EC-08/12) — ターンループとセッション排他。構成節で記述(単純なためADR化しない)

## Phase 2: 判断結果 (2026-07-07)

- ADR-01 UI/スタック = **Tauri 2 + TypeScript** [承認済み]
- ADR-02 推論接続 = **OpenAI互換API標準、推奨エンジンOllama** [承認済み]
- ADR-03 全体構成 = Tauri単一プロセス [提案]
- ADR-04 想起 = **ハイブリッド(類似度+新しさ+重要度)** [承認済み]
- ADR-05 人格表現 = **数値軸+自由記述** [承認済み]
- ADR-06 後処理タイミング = セッション確定後の非同期バッチ+起動時リカバリ [提案]
- ADR-07 永続化 = SQLite単一DB(WAL) [提案]
