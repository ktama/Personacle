# design-notes: Personacle 設計 作業メモ

入力: docs/requirements.md v0.1 (FR-01〜19, NFR-01〜07, EC-01〜12)

## Phase 1: 論点リスト

1. **UI形態と技術スタック** (要件 未解決9-1, NFR-07, 将来公開) — デスクトップアプリ / ローカルWeb / スタック選定
2. **推論エンジンとの接続方式** (未解決9-2, FR-16, NFR-01/02) — 接続プロトコルの標準と推奨エンジン
3. **全体構成** (A-1) — 単一プロセスかフロント/バック分離か。ADR-01に従属
4. **データ永続化方式** (FR-17, NFR-04/05, A-2) — SQLite / ファイル群 / 併用
5. **記憶の想起方式** (FR-09, NFR-01/04, EC-06) — 埋め込み類似検索 / 全文検索 / ハイブリッド
6. **記憶生成・人格更新のタイミング** (FR-08/12, NFR-01, EC-03) — 逐次 / セッション終了時 / バックグラウンド
7. **人格プロファイルの表現** (FR-12/13, 未解決9-4) — 数値パラメータ / 自由記述 / ハイブリッド。履歴の持ち方(追記イベント+現在値)
8. **自律会話の実行制御** (FR-14/15/19, EC-08/12) — ターンループとセッション排他。構成節で記述(単純なためADR化しない)

## Phase 2: 判断結果 (2026-07-07)

- ADR-01 UI/スタック = **Tauri 2 + TypeScript** [承認済み]
- ADR-02 推論接続 = **OpenAI互換API標準、推奨エンジンOllama** [承認済み]
- ADR-03 全体構成 = Tauri単一プロセス [提案]
- ADR-04 想起 = **ハイブリッド(類似度+新しさ+重要度)** [承認済み]
- ADR-05 人格表現 = **数値軸+自由記述** [承認済み]
- ADR-06 後処理タイミング = セッション確定後の非同期バッチ+起動時リカバリ [提案]
- ADR-07 永続化 = SQLite単一DB(WAL) [提案]

## v0.2 追補設計 作業メモ (2026-07-08 開始)

入力: docs/requirements.md v2.0 draft (FR-20〜35, NFR-08〜09, EC-13〜20)。要件IDあり、Phase 0 合格。

### Phase 1 (v0.2): 論点リスト

1. **話しかけのセッション扱いと生成方式** (FR-21, NFR-09, EC-13/14, 要件9-8) — 画面オープン時にセッションを開始するか、応答があるまで仮発話とするか。無応答セッションの後処理の扱い
2. **時間経過の認識方式** (FR-20, EC-18) — 経過時間をコアで区分ラベル化して渡すか、生の日時をLLMに渡すか
3. **記憶統合のトリガーと類似判定** (FR-22/23, EC-15/16/20, 要件9-9) — 統合の契機(後処理組み込み/件数閾値/起動時)と、統合候補の見つけ方(埋め込み/LLM/全文検索)
4. **ムードの表現形式と回帰の実現** (FR-24/25, 要件9-10) — 軸数・値域、平常への回帰を常駐処理なしでどう実現するか
5. **日記の生成タイミング** (FR-26/27, EC-17) — 日付確定後にまとめて生成か、セッションごとに当日分を再生成か
6. **グループチャットの話者選択と連鎖発話** (FR-31〜34, NFR-01, 要件9-12) — 「文脈に基づく1体」の選択方式。自律会話でLLM選択を却下したADR-08との整合
7. **停滞検出と話題転換の方式** (FR-35, 要件9-11) — 検出(埋め込み/LLM/表層一致)と転換の実現、履歴への記録方法
8. **時系列グラフ・関係図の描画手段** (FR-29/30, NFR-08) — 描画ライブラリ導入か自作SVGか(ローカル完結のためCDN不可、バンドル必須)

記憶検索(FR-28)は既存Storage+一覧画面の拡張で自明のためADR化しない(構成節で記述)。

### Phase 2 (v0.2): 判断結果 (2026-07-08)

発注者確認済み(4件、いずれも推奨案で承認):

- ADR-13 ムード表現 = **1軸(快-不快 -100〜+100)+LLM生成の状態ラベル。半減期による遅延回帰計算** [承認済み]
- ADR-15 話者選択 = **LLM選択+コア補正(連続選択制限・未発話者優先・不一致時ラウンドロビンにフォールバック)** [承認済み]
- ADR-14 日記生成 = **セッション後処理の末尾で当日分を都度更新(デバウンス付き)** [承認済み]
- ADR-12 統合トリガー = **セッション後処理に組み込み(新規記憶の周辺で類似クラスタ検出)** [承認済み]

技術判断として提案ステータスで記載:

- ADR-10 話しかけ = 画面オープンでセッション開始+話しかけを通常発話として保存。ユーザー発話ゼロのセッションは後処理スキップ [提案]
- ADR-11 時間経過 = コアが経過時間を区分ラベル化してプロンプト注入(LLMに日時計算をさせない) [提案]
- ADR-16 停滞検出 = 埋め込み類似度(同一話者の直近発話と連続高類似)+システム発話による話題転換 [提案]
- ADR-17 グラフ描画 = 自作SVG(依存追加なし。既存フロントがバニラTSのため整合) [提案]

### Phase 3-5 (v0.2): 完了記録 (2026-07-08)

- 成果物: docs/design.md v2.0 (draft)。トレーサビリティ表は本文9章が正(本メモの骨格は役目を終えた)
- トレーサビリティ機械検査: 要件書64 ID(FR35+NFR9+EC20)と設計書9章の行が双方向で完全一致(comm で確認)
- 要件捏造検査: 追加要素の根拠 — last_greeting_at=EC-14、mood_*/mood_event=FR-24/25、diary=FR-26/27、consolidated_into=FR-22/23、speaker_selecting event=ADR-15の遅延中フィードバック、移行時の初回統合ジョブ=FR-22(既存記憶も統合対象)。全て要件IDまたはADRに根拠あり
- 実装過剰検査: 関数本体・ファイル構成・命名規約なし。減衰式・閾値は振る舞いの仕様として記載(実装指定ではない)

- 承認待ちADR: ADR-10(話しかけのセッション扱い), ADR-11(時間経過ラベル化), ADR-16(停滞検出), ADR-17(自作SVG) → 設計書11章 D-6
